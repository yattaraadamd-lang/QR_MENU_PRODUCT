🔒 TENANT ISOLATION

Önce (Güvensiz):
  const { businessId } = await request.json(); // ❌ Client'tan alınıyor
  const products = await prisma.product.findMany({ where: { businessId } });

Sonra (Güvenli):
  const authResult = await requireAdmin(); // ✅ Session'dan alınıyor
  const businessId = getBusinessIdFromSession(authResult.session);
  const products = await prisma.product.findMany({ where: { businessId } });

Kazanç:
  ✓ Cross-tenant data leak engellendi
  ✓ Privilege escalation engellendi
  ✓ IDOR (Insecure Direct Object Reference) engellendi

🔒 INPUT VALIDATION

Önce (Güvensiz):
  if (!name || !price) { // ❌ Manuel, eksik validation
    return error;
  }

Sonra (Güvenli):
  const validation = validateBody(createProductSchema, body); // ✅ Zod
  if (!validation.success) {
    return NextResponse.json({ error: validation.error }, { status: 400 });
  }

Kazanç:
  ✓ Type safety
  ✓ Comprehensive validation
  ✓ Better error messages
  ✓ SQL injection koruması

🔒 RESOURCE OWNERSHIP

Önce (Güvensiz):
  const product = await prisma.product.findUnique({ where: { id } }); // ❌ Ownership check yok

Sonra (Güvenli):
  const product = await prisma.product.findFirst({
    where: { id, businessId }, // ✅ Tenant check
  });

Kazanç:
  ✓ Unauthorized access engellendi
  ✓ Data leak engellendi

🔒 TRANSACTION SAFETY

Önce (Race Condition Riski):
  const order = await prisma.order.create({ ... }); // ❌ Separate operations
  await prisma.table.update({ ... });
  await prisma.notification.create({ ... });

Sonra (Atomic):
  const result = await prisma.$transaction(async (tx) => { // ✅ Transaction
    const order = await tx.order.create({ ... });
    await tx.table.update({ ... });
    await tx.notification.create({ ... });
    return order;
  });

Kazanç:
  ✓ Data consistency
  ✓ No partial updates
  ✓ Rollback on error

🔒 MINIMAL SELECT

Önce (Over-fetching):
  include: { // ❌ Tüm alanlar ve ilişkiler
    items: { include: { product: true } },
    table: true,
    waiter: true,
  }

Sonra (Minimal):
  select: { // ✅ Sadece gerekli alanlar
    id: true,
    totalPrice: true,
    items: { select: { productName: true, quantity: true } },
  }

Kazanç:
  ✓ Reduced bandwidth
  ✓ Faster queries
  ✓ Less memory usage
  ✓ No sensitive data leak

⚠️  1. API RESPONSE DEĞİŞİKLİKLERİ

Minimal select kullanıldığı için response'lar değişti.

Önce:
  {
    "orders": [{
      "items": [{ "product": { /* tüm alanlar */ } }],
      "table": { /* tüm alanlar */ }
    }]
  }

Sonra:
  {
    "orders": [{
      "items": [{ "productName": "...", "quantity": 2 }],
      "table": { "tableNumber": "1" }
    }]
  }

⚠️  2. QUERY PARAMETER DEĞİŞİKLİKLERİ

GET /api/orders artık businessId query parameter'ı kabul etmiyor.

Önce: /api/orders?businessId=xxx ✅
Sonra: /api/orders?businessId=xxx ❌ (ignore edilir)

⚠️  3. ERROR RESPONSE DEĞİŞİKLİKLERİ

Validation error'ları daha detaylı.

Önce: { "error": "Invalid input" }
Sonra: { "error": "items.0.quantity: Number must be greater than 0" }
