  ⚠️  MANUEL YAPILMASI GEREKENLER
═══════════════════════════════════════════════════════════════════════════════

1. 🔑 PRODUCTION SECRET ROTATION
   
   Eğer .env.production daha önce commit edildiyse:
   
   a) Yeni NEXTAUTH_SECRET oluştur:
      node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   
   b) Database şifresini değiştir:
      ALTER USER your_user WITH PASSWORD 'new_strong_password';
   
   c) Hosting platformunda environment variables güncelle:
      - Vercel: Project Settings → Environment Variables
      - AWS: Systems Manager Parameter Store
      - Heroku: Config Vars

2. 🧹 GIT HISTORY TEMİZLİĞİ (Opsiyonel ama Önerilir)
   
   ⚠️  UYARI: Önce yedek alın!
   
   # BFG Repo-Cleaner ile (önerilir):
   bfg --delete-files .env.production
   git reflog expire --expire=now --all
   git gc --prune=now --aggressive
   git push origin --force --all

3. 🚀 PRODUCTION RATE LIMITING
   
   In-memory rate limiting sadece development içindir.
   Production'da Redis kullanın:
   
   # Vercel için:
   npm install @upstash/ratelimit @upstash/redis
   
   # Diğer platformlar için:
   npm install ioredis rate-limiter-flexible

4. 📊 MONITORING KURULUMU
   
   Production'da izleyin:
   - Başarısız login denemeleri
   - Rate limit aşımları
   - Geçersiz token kullanımı
   - Yetkisiz erişim denemeleri
   
   Önerilen araçlar:
   - Sentry (Error tracking)
   - Datadog (Infrastructure monitoring)
   - Vercel Analytics
